Menubar

The Menubar contains the following choices:

  • File
    • Print to printer/file Print
    • Save the diagram as an image with Save Image You can choose between XPM/BMP/PNG og EPS. The default is PNG. Every format exept EPS gives a screendump (WYSIWYG). EPS is dynamic.
    • Create your own positionfilter with Change filter
    • Quit the program with Quit
  • Preferences
    • The user can save the actual set of window -size/pos, position, type, model by Save preferences. The saved values will be reused next time T-series is started or when the Reset preferences option is used.
    • The prefernces are saved automatically at exit when the Save at exit option is on.
    • Show postitions (DIANA) Displays all position names at the DIANA map in case of linking between DIANA and T-series. This option can distract the user at large datasets but is ok for small sets...
    • Show active position (DIANA) Shows just the name of the diagram position on the map (recommended)
    • Vis timemark shows a gray line at the current time and a red line at the DIANA-time.
  • Help
    • Manual Shows this document
    • About T-series..Version number etc.

Linking to DIANA


T-series can link up to a diana-instance if DIANA is open to such linking. The linking is controlled by an external application (coserver) which is socket based. coserver starts automtically when needed. The user can couple by T-series and DIANA by activate the in both applications.

After the linking the following happens:

  • T-series sends a full list of all positions to DIANA. Those will show up in the DIANA map as icons if the Show icons (DIANA) option is on. T-series Icons look like:
  • BY choosing a position in DIANA, T-series will show the belonging Diagram.
  • By clicking T-series will send a pointer to DIANA to find the position of the actual diagram.

Wdb Connection

From version 5.0 tseries is able to connect to a WDB database and get data interpolated on demand.
The Tab name is similar to the database-server which is used in the connection.
The user interface is coordinate based, you can choose coordinates by the lat/lon sliders, write them directly into the Lat/Lon textfields or click into any given place in the diana map to get the diagram for the specific coordinates. Remember to hold down the SHIFT button when you are in the profet-edit modus

All the modelnames,times and coordinate borders are selected from the database. When the database-tab is disabled, the connection to the database failed for some reason.

CACHE
The database interpolates data for you on demand, for this, the server has to read all the fields nessecary to generate your diagram. If this is to slow (default > 1.5s) the cache button becomes active. By pushing the cache button you send a request to the server to take all your fields into memory - speeding up the interpolation. When you have cached a fieldset, the fieldset will be fast for anyone who is using it. The caching process is running in background and takes up to one minute. Its finished when the red blinking icon vanishes.

BOOKMARS
One can easyly define bookmarks in tseries. By pushing the button one can add new folder to the Bookmarks. Just doubleclick on the NEW folder and rename it. Now one can drag positions from the other folders there. Rename the bookmarks as you wish. There are some distributed folders with given positions. These can be edited, but will be restored after restart. Private created folders will be stored locally ($HOME/.tseries/bookmarks.def) All positions can be dragged into trash for removal. Empty private folders are automatically removed after restart. Any choosen position is tracked under the RECORD folder. Drag from there into a private folder if you want to track the actual position.
